📄 1. PRD (Product Requirements Document)
1.1 Product Name

(Working Title): RideHub Campus

1.2 Objective

Create a centralized vehicle rental discovery and coordination platform for a college ecosystem that:

Aggregates local vendors
Displays vehicle inventory
Enables booking requests
Facilitates direct vendor-user communication
1.3 Scope (Strict)
IN-SCOPE
Vendor onboarding
Vehicle listings
Search & filter
Booking request system
Contact integration (call/WhatsApp)
Admin verification system
OUT-OF-SCOPE (DO NOT BUILD)
Payments
Real-time inventory sync
Advanced analytics
AI features
Multi-city scaling
1.4 User Types
1. Student (End User)
Browses vehicles
Requests booking
Contacts vendor
2. Vendor
Manages listings
Responds to requests
3. Admin (You)
Verifies vendors
Maintains data quality
1.5 Core Features
F1: Vehicle Discovery
List all vehicles
Filters:
Type (bike/scooter/car)
Price range
Distance (optional)
Sorting:
Price (low → high)
Rating (future-ready)
F2: Vehicle Detail Page
Images (required)
Price
Vendor name
Contact options
Availability status
Last updated timestamp
F3: Booking Request
Input:
Name
Phone
Duration
Output:
Stored request
Vendor notified (manual)
F4: Vendor Panel (Basic)
Add/edit/delete vehicles
Toggle availability
Update pricing
F5: Admin Controls
Approve/reject vendors
Remove inactive listings
Edit incorrect data
1.6 Success Metrics
of active vendors
of listings
of booking requests per day
Vendor response rate
